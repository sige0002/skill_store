# Rosbag → LeRobot Dataset 変換

実機で記録した rosbag（mcap）を LeRobot データセットに変換する際の定型手順と、実際にハマった落とし穴集。

## 変換パイプラインの設計

1. **入力を mcap に正規化する** — ROS 1 `.bag` / sqlite3 形式は `rosbags-convert` 等で事前に mcap へ変換し、変換器本体は mcap だけを扱う
2. **必須トピックの定義** — 変換に必要なトピック（関節状態・画像・アクション）を明示的にリスト化し、欠けている bag は早期にエラーにする
3. **時刻同期とトリム**:
   - **全必須トピックが揃った時刻を start にする**（購読ラグで序盤は欠けるため）
   - **最も早く終わるトピックの時刻で trim する**（終端の不揃いを切る）
   - フレームのペアリングは header.stamp 基準で最近傍マッチングする
4. **TRANSIENT_LOCAL の古いフレーム除去** — durability が TRANSIENT_LOCAL のトピックは記録開始前の古いメッセージが混入する。header 時刻が記録開始より一定以上古いフレームは閾値で捨てる

## 検証（変換後に必ず行う）

- **画像 shape の確認** — 画像は channel-first `(3, H, W)`（channel=3）になっているか。ここが崩れていると LeRobot の学習がそのまま失敗する
- **stats の突合** — 正規化統計量（mean/std/min/max）の計算方法を LeRobot 本家実装と突き合わせる。列の並び・配置は HuggingFace 上の参照データセットと比較する
- **キー名のマッピング** — データセット側のキー名（例: `right_arm_joint_1_rad`）とロボット実装側のキー名（例: `right_joint_1.pos`）が異なる場合、デプロイ時のリネームマッピングを変換時に文書化しておく
- **エンドツーエンド検証** — 変換したデータセットで実際に学習（小規模でよい）が回ることを確認して初めて完了とする

## 運用の知恵

- **プレビュー mp4 はキャッシュする** — 目視確認用に image topic を ffmpeg で mp4 化する場合、変換済みの結果を残して再利用する（毎回の再変換は遅い）
- **A/B 検証** — 変換ロジックを修正したら、修正前後のコードで同一の実機 bag を変換し、別のエージェント/レビュアー（codex 等）に時間比較と齟齬チェックをさせると確実
- **大容量データは gitignore** — bag・変換出力・データセットはリポジトリに乗せない（github-publish スキル参照）
