# sync-docs

日本語ドキュメント（正本）から、対応する英語ミラーを再生成して同期するスキル。

## 正本ルール

- **日本語が正本。** 内容の編集は日本語側で行う。**著者も Claude も、作業として日本語の正本を直接編集してよい**（むしろ推奨）。編集後にこのスキルで英語を再生成する。
- **英語は自動生成ミラー。** 英語ファイルは手で編集しない。
- **同期は日本語 → 英語の一方向。** このスキルの同期処理では、英語から日本語を逆生成しない／日本語ファイルを書き換えない。「Claude が日本語を触ってはいけない」という意味ではなく、「翻訳の向きが一方向」という意味。

## 対応関係（日本語ソース → 英語ミラー）

- `CLAUDE.ja.md` → `CLAUDE.md`
- `README.ja.md` → `README.md`
- `docs/specs/ja/<name>.md` → `docs/specs/en/<name>.md`

一般則:
- ルート直下などの `*.ja.md` は、`.ja` を外したパスが英語ミラー。
- `docs/specs/ja/` 配下のファイルは、`docs/specs/en/` の同名ファイルが英語ミラー。
- したがって「日本語ソース」とは、`*.ja.md` または `docs/specs/ja/` 配下のファイルを指す。

## 手順

1. **同期対象の日本語ソースを決める**（優先順）:
   - 引数でファイル／パスが指定されていれば、それ。
   - `--staged` なら、`git diff --cached --name-only` のうち日本語ソース。
   - それ以外は、`git status --porcelain`（staged + unstaged + untracked）の変更分から日本語ソースを抽出。
   - `all`（または「全部同期して」）なら、リポジトリ内の全日本語ソース（`git ls-files '*.ja.md' 'docs/specs/ja/**'`）。
   - 該当なしなら「同期対象なし」と報告して終了。

2. **各日本語ソースについて、対応する英語ミラーを現在の日本語に忠実に再生成する**:
   - 見出し・順序・表・箇条書き・コードブロック・引用などの構造をそのまま保つ。
   - 散文だけ翻訳する。コード・識別子・ファイルパス・API ルート・コマンド名・固有名詞（ROS 2, MCAP, dora, WebRTC など）は訳さない。
   - リポジトリ内リンクは英語側に張り替える（`README.ja.md` → `README.md`、`docs/specs/ja/x.md` → `docs/specs/en/x.md`、`../ja/` ↔ `../en/` も対応させる）。
   - **追加も削除もしない。** 翻訳のみ（新しい設計・例・意見を足さない）。
   - 英語ミラーの 1 行目を、必ず次のバナーにする（`<ja-path>` は実際のソースパス）:
     ```
     <!-- AUTO-GENERATED from <ja-path>. Do not edit by hand — edit the Japanese source and run /sync-docs. -->
     ```

3. **安全策**:
   - 同期処理では日本語ソースを書き換えない（＝英語から日本語を逆生成しない）。日本語の編集は通常の作業として別途行うこと。
   - 既存の英語ミラーの 1 行目がバナーでない場合、手編集が混じっている可能性があるので、上書き前にユーザーへ警告する。

4. **報告**: 作成／更新した英語ミラー、およびスキップ／警告したものを簡潔にまとめる。

## メモ

- これはオンデマンド実行。Claude Code のフックは、ユーザーがエディタで直接行った編集を検知できないため、日本語を編集した後に `/sync-docs` を実行する。
- コミット時に自動化したい場合は、git の `pre-commit` フックから `claude -p "/sync-docs --staged"` を呼ぶ（非対話の認証が必要）。
