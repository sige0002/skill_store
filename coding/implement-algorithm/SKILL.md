# アルゴリズム実装スキル

以下のリクエストに記述されたアルゴリズムを実装する：

**Request:** $ARGUMENTS

## ワークフロー

### Phase 1: 論文分析
**ultrathink** を用いてアルゴリズムを深く分析する。

- 論文から中核アルゴリズム・方程式・疑似コードを特定する
- すべての数式演算、データ構造、パラメータを列挙する
- 入出力仕様、計算量（時間・空間）の主張を記録する
- 数値安定性の懸念（オーバーフロー、アンダーフロー、精度）を洗い出す
- 論文記載の指標・データセット・ベースライン比較を抽出する

### Phase 2: 設計
- 数式表記をコード構造にマッピングする
- 適切なデータ構造（配列、木、グラフ、ヒープなど）を選ぶ
- モジュール構成を計画する：コアアルゴリズム、ユーティリティ、テスト、ベンチマーク
- 再利用可能なライブラリ関数（numpy、scipy など）とカスタム実装を区別する

### Phase 3: 環境セットアップ
```bash
uv venv .venv && source .venv/bin/activate
uv pip install numpy scipy pytest  # 必要に応じて追加
```

### Phase 4: 実装
複雑なアルゴリズム部分には **ultrathink** を使う。

- 論文の疑似コードに忠実に、段階的に実装する
- 方程式番号やアルゴリズム行番号を参照するインラインコメントを付ける
  ```python
  # Eq. (3): Update step with momentum term
  v_t = beta * v_{t-1} + (1 - beta) * grad
  ```
- 型ヒントと、パラメータ説明を含む docstring を付ける
- サブルーチンにはヘルパー関数を実装する
- エッジケースを扱う：空入力、単一要素、非常に大きい入力

### Phase 5: テスト
- 論文に記載された結果（表・図）からテストケースを作成する
- 各サブルーチンにユニットテストを追加する
- エッジケースと境界条件をテストする
- 既知の正しい出力や参照実装と照合する：
  ```bash
  uv run pytest tests/ -v
  ```

### Phase 6: ベンチマーク
- 入力サイズを変えて実行時間を測定し、計算量の主張を検証する
- 利用可能ならベースライン実装と比較する
- ホットスポットをプロファイリングする：
  ```bash
  uv run python -m cProfile -s cumtime algorithm.py
  ```
- レポートする：入力サイズ、実行時間、メモリ使用量、論文との精度差

### Phase 7: ドキュメント
- 要点をまとめる：アルゴリズム名、論文参照、計算量、使い方
- 論文からの逸脱とその理由があれば明示する
- 既知の制限や前提条件を列挙する

## ガイドライン
- 数値計算コード：ベクトル化操作には `numpy`/`scipy` を使う
- 早まった最適化は避ける — まず正しさ、その後プロファイル
- コメントや docstring で論文を明示的に参照する
